# 自动化监测与飞书推送功能说明

## 功能概述

核心功能：自动化内容监测与飞书实时推送。

## 已实现的功能

### 1. 抓取引擎 (`lib/crawler.ts`)

#### Apify API 集成
- ✅ 基于 Apify API 的脚本逻辑
- ✅ 支持 Mock 数据模式 (未配置 API Key 时)
- ✅ 真实 API 调用框架 (待填入 API Key)
- ✅ Actor 运行状态监控

- ✅ 大小写不敏感匹配

#### Mock 数据生成
- ✅ 随机生成 1-3 条模拟内容
- ✅ 自动组合车型名和关键词生成标题
- ✅ 模拟互动数据 (点赞、评论、收藏)
- ✅ 随机发布时间 (最近 7 天内)

### 3. 自动监测 API (`app/api/monitor/route.ts`)

#### POST `/api/monitor`

**功能流程**:
```
1. 查询所有 driving 状态的申请
2. 遍历每个申请:
   a. 解析关键词
   b. 识别平台
   c. 调用 Apify API (或 Mock)
   d. 遍历抓取结果:
      - 匹配车型名 + 关键词
      - 检查是否已存在
      - 存入 Contents 表 (status: 'auto')
      - 发送飞书通知
3. 返回统计结果
```

**请求**:
```bash
POST /api/monitor
```

**响应**:
```json
{
  "success": true,
  "message": "监测完成，共匹配 3 条新内容",
  "data": {
    "total": 5,
    "matched": 3
  }
}
```

## 配置说明

### 环境变量

在 `.env.local` 中配置:

```bash
# Apify API
APIFY_API_KEY=apify_api_xxx
APIFY_ACTOR_ID=your-actor-id

# 飞书 Webhook
FEISHU_WEBHOOK_URL=https://open.feishu.cn/open-apis/bot/v2/hook/xxx
```

### Apify 配置步骤

1. 注册 Apify 账号: https://apify.com
2. 创建或选择 Actor:
   - 小红书抓取 Actor
   - 抖音抓取 Actor
3. 获取 API Key:
   - 进入 Settings → Integrations
   - 复制 API Token
4. 获取 Actor ID:
   - 进入 Actor 详情页
   - URL 中的 ID 或在 API 标签中查看

### 飞书配置步骤

1. 创建飞书机器人:
   - 进入飞书群组
   - 点击"设置" → "群机器人"
   - 点击"添加机器人" → "自定义机器人"
2. 配置机器人:
   - 名称: KOC 内容监测
   - 描述: 自动推送 KOC 内容匹配通知
3. 获取 Webhook:
   - 复制 Webhook 地址
   - 格式: `https://open.feishu.cn/open-apis/bot/v2/hook/...`

## 技术架构

### 抓取流程
```
Apify API
    ↓
Actor 执行 (小红书/抖音爬虫)
    ↓
Dataset 存储
    ↓
获取数据 (标题、正文、互动数据)
    ↓
关键词匹配
    ↓
存入 Contents 表
    ↓
飞书推送
```

### Mock 流程 (无 API Key)
```
Mock 数据生成
    ↓
模拟标题 (包含车型名+关键词)
    ↓
模拟互动数据
    ↓
关键词匹配 (100% 匹配)
    ↓
存入 Contents 表
    ↓
飞书推送 (或打印日志)
```

## 常见问题

### Q1: Mock 数据一定会匹配成功吗?
A: 是的。Mock 数据生成器会自动组合车型名和关键词到标题中，确保 100% 匹配。

### Q2: 真实 API 需要多久执行完?
A: 取决于 Apify Actor 的运行时间,通常 30 秒到 5 分钟不等。

### Q3: 飞书推送失败会影响主流程吗?
A: 不会。飞书推送失败只会打印错误日志,不影响数据保存。

### Q4: 如何配置定时任务?
A: 可使用 Vercel Cron Jobs 或第三方服务 (如: EasyCron、cron-job.org)。

### Q5: 支持哪些平台?
A: 当前支持小红书、抖音、微博。可扩展其他平台。
