## 🎯 完整业务流程

```
┌─────────────────────────────────────────────────────┐
│ 方式 1: 自动监测                                      │
└─────────────────────────────────────────────────────┘

1. 用户提交申请 (/apply)
   ↓
2. 管理员审核通过 → 发车 (status: driving)
   ↓
3. 自动监测任务执行 (手动或定时)
   - POST /api/monitor
   ↓
4. 抓取 KOC 平台内容
   - 调用 Apify API (或 Mock)
   ↓
5. 关键词匹配
   - 必须包含: 车型名 + 关键词
   ↓
6. 匹配成功
   - 存入 Contents 表 (status: 'auto')
   - 发送飞书通知 (🤖 自动监测)
   ↓
7. 飞书群收到卡片消息


┌─────────────────────────────────────────────────────┐
│ 方式 2: 手动关联                                      │
└─────────────────────────────────────────────────────┘

1. 管理员进入申请详情页
   ↓
2. 粘贴内容链接 + 点击"关联"
   ↓
3. Mock API 获取内容数据
   ↓
4. 存入 Contents 表 (status: 'manual')
   ↓
5. 发送飞书通知 (👤 手动关联)
   ↓
6. 飞书群收到卡片消息
```
