# 申请表单功能说明

## 功能概述

用户端借车申请表单页面 `/apply`，参考金数据的简洁大方 UI 风格设计。

## 已实现的功能

### 1. 页面路由
- **路径**: `/apply`
- **文件**: `app/apply/page.tsx` (服务端组件)
- **表单组件**: `app/apply/ApplyForm.tsx` (客户端组件)

### 2. 核心字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 姓名 | 文本 | ✅ | 申请人真实姓名 |
| 手机号 | 电话 | ✅ | 11位手机号，带格式验证 |
| 车型 | 下拉选择 | ✅ | 从 Cars 表动态读取，显示库存 |
| 开始日期 | 日期 | ✅ | 不能早于今天 |
| 结束日期 | 日期 | ✅ | 必须晚于开始日期 |
| 平台链接 | URL | ✅ | 小红书或抖音主页链接 |
| 内容关键词 | 文本 | ✅ | 引导式设计，带提示 |

### 3. 内容关键词引导设计

#### UI 提示语
```
请输入计划包含的关键词
例如：极氪001 露营、特斯拉 自驾游、Model Y 城市通勤
```

#### 引导文案
```
💡 提示：发布内容时包含这些词，系统能更精准地识别您的作品
建议格式：车型名 + 使用场景，多个关键词用空格或逗号分隔
```

### 4. 数据流程

1. **前端提交** → 2. **API 验证** → 3. **存入数据库** → 4. **返回成功**

```
用户填写表单
    ↓
客户端表单验证
    ↓
POST /api/applications
    ↓
服务端验证（手机号、日期、车辆库存等）
    ↓
插入 Applications 表（status: pending）
    ↓
返回申请编号
    ↓
显示成功弹窗
    ↓
3秒后跳转首页
```

### 5. API 端点

**路径**: `/api/applications`
**方法**: `POST`
**文件**: `app/api/applications/route.ts`

#### 请求体
```json
{
  "name": "张三",
  "phone": "13800138000",
  "car_id": "uuid",
  "start_date": "2026-03-15",
  "end_date": "2026-03-20",
  "platform_url": "https://www.xiaohongshu.com/user/xxx",
  "keywords": "极氪001 露营"
}
```

#### 成功响应
```json
{
  "success": true,
  "message": "申请提交成功",
  "data": {
    "id": "full-uuid",
    "application_number": "ABC123"
  }
}
```

#### 错误响应
```json
{
  "error": "错误信息"
}
```

## 测试步骤

### 1. 启动项目
```bash
cd koc-management
npm run dev
```

### 2. 访问页面
打开浏览器访问: `http://localhost:3000/apply`

### 3. 填写表单
- 输入姓名：张三
- 输入手机号：13800138000
- 选择车型：任意有库存的车型
- 选择日期：今天之后的日期范围
- 输入平台链接：https://www.xiaohongshu.com/user/test
- 输入关键词：极氪001 露营

### 4. 提交测试
点击提交按钮，观察：
- ✅ 表单验证是否生效
- ✅ 提交按钮是否显示加载状态
- ✅ 是否显示成功弹窗
- ✅ 申请编号是否生成
- ✅ 是否自动跳转

### 5. 数据验证
在 Supabase 后台查看 `applications` 表，确认数据已插入。
